# QA 検出基準

## エラーハンドリングとログ

| 基準 | 判定 |
|------|------|
| エラーの握りつぶし（空のcatch） | REJECT |
| ユーザー向けエラーメッセージが不明確 | 修正が必要 |
| システム境界でのバリデーション欠如 | 警告 |
| 新しいコードパスにデバッグログがない | 警告 |
| ログへの機密情報の出力 | REJECT |

## 保守性

| 基準 | 判定 |
|------|------|
| 関数/ファイルが複雑すぎる（追いにくい） | 警告 |
| 重複コードが多い | 警告 |
| 命名が不明確 | 修正が必要 |

## 技術的負債

| パターン | 判定 |
|---------|------|
| TODO/FIXMEの放置 | 警告 |
| 理由なしの @ts-ignore, @ts-expect-error | 警告 |
| 理由なしの eslint-disable | 警告 |
| 非推奨APIの使用 | 警告 |
